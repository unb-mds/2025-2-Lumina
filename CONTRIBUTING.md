# Guia de Contribuição - Lumina 💡

Obrigado pelo interesse em contribuir com o **Lumina**! 🎉

Este documento contém as diretrizes para garantir que sua contribuição seja processada de forma rápida e eficiente. Por favor, leia com atenção antes de começar.

---

## 🚀 Como Contribuir

Para contribuir com o projeto, siga o fluxo de trabalho abaixo:

### 1. Encontre ou Crie uma Issue

Antes de escrever qualquer código:

* Verifique as **Issues Abertas** para ver se alguém já está trabalhando no problema ou funcionalidade.
* Se for algo novo, crie uma **Issue** descrevendo o problema ou a sugestão de melhoria. Aguarde a aprovação ou discussão da equipe antes de começar.

### 2. Faça um Fork e Clone o Repositório

1.  Faça um **Fork** deste repositório para a sua conta.
2.  Clone o repositório para sua máquina local:

```bash
git clone [https://github.com/SEU_USUARIO/2025-2-Lumina.git](https://github.com/SEU_USUARIO/2025-2-Lumina.git)
cd 2025-2-Lumina
```

### 3. Crie uma Branch

Crie uma branch específica para a sua contribuição. Use nomes descritivos e siga o padrão:

* `feat/nome-da-funcionalidade` (para novas funcionalidades)

* `fix/nome-da-correcao` (para correção de bugs)

* `docs/nome-da-mudanca` (para documentação)

* `chore/nome-da-tarefa` (para manutenção ou configurações)

Exemplo:

```bash
git checkout -b feat/adicionar-login
```

### 4. Faça Commits Semânticos

Use mensagens de commit claras e padronizadas (em inglês ou português, conforme a decisão da equipe, mas mantenha a consistência).
Exemplos:

* `feat: Adiciona endpoint de login`

* `fix: Corrige erro no crawler do G1`

* `docs: Atualiza instruções no README`

### 5. Envie suas alterações (Push)

```bash
git push origin nome-da-sua-branch
```

### 6. Abra um Pull Request(PR)

1. Vá até o repositório original no GitHub.

2. Clique em **Compare & pull request**.

3. Preencha o template do PR com detalhes sobre o que foi feito.

4. Aguarde a revisão da equipe. Fique atento aos comentários!


Boas contribuições! 🚀
